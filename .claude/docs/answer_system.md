# Система множественных ответов

## Расположение

- Парсер: `lib/core/utils/answer_parser.dart`
- Конвертер карточек: `lib/feature/test_detail/data/converters/card_converter.dart`
- Импорт из файла: `lib/core/services/csv_import_service.dart`
- Диалог импорта: `lib/feature/test_detail/presentation/csv_import_dialog.dart`

## Как хранятся ответы

В БД ответ хранится как **единая строка** в поле `answer`. Множественные варианты разделяются символами-разделителями прямо в строке (например `"кот | кошка"`).

## Парсинг (AnswerParser.parse)

При чтении из БД (`CardConverter.fromDto`) строка ответа прогоняется через `AnswerParser.parse()`, который возвращает `List<String>`.

Поддерживаемые форматы (в порядке приоритета):
1. `"a | b | c"` — основной формат через `|`
2. `"a // b // c"` — альтернативный формат через `//` (AI-friendly, добавлен потому что нейросети плохо копируют `|`)
3. `"a (b)"` — legacy формат со скобками
4. `"a - b"` — legacy формат с дефисом

Максимум **3 варианта** ответа.

## Форматирование (AnswerParser.format)

При сохранении `List<String>` обратно в строку используется `AnswerParser.format()` → соединяет через ` | `.

## Поток данных при создании карточки

1. **Ручное создание / импорт** → `back` сохраняется в БД как есть (без нормализации)
2. **Чтение из БД** → `AnswerParser.parse()` разбирает строку на варианты
3. **Редактирование** → `AnswerParser.parse()` вызывается при сохранении, затем `AnswerParser.format()` нормализует в формат с `|`

То есть нормализация в единый формат `|` происходит только при редактировании существующей карточки, но не при первичном создании. Это не проблема, т.к. парсер понимает все форматы при чтении.

## Импорт из файла

Диалог импорта (`CsvImportDialog`) показывает:
- Описание формата файла (CSV/TXT/TSV, 2 столбца)
- Подсказку про множественные ответы через `|` или `//`
- Пример в monospace-блоке
- Настраиваемый разделитель столбцов (по умолчанию `;`)
