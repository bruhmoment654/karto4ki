# Скоринговый алгоритм подмешивания (ScoringMixupService)

## Расположение (планируемое)

`lib/feature/tinder_test/domain/mixup/scoring_mixup_service.dart`

## Назначение

Замена классического алгоритма. Вместо двух отдельных веток сортировки использует единую скоринговую модель, которая одновременно учитывает все факторы: сложность, забывание, новизну и частоту ошибок.

## Получение кандидатов

Идентично классическому алгоритму — вынесено в общий `MixupCandidateLoader`:
1. Группы текущего теста → все тесты групп → исключить текущий
2. Загрузить карточки, отфильтровать дубликаты по нормализованному ключу
3. Загрузить статистику

## Скоринговая модель

Для каждого кандидата вычисляется `score` (0.0–1.0+). Чем выше — тем приоритетнее.

```
score = W_streak  * streakFactor
      + W_decay   * decayFactor
      + W_new     * newFactor
      + W_freq    * frequencyFactor
      + noise
```

### Факторы

#### streakFactor (вес 0.35)

Приоритет вопросов с отрицательным streak (ошибки подряд).

```dart
streakFactor = streak < 0
    ? clamp(-streak / 5, 0, 1)
    : 0;
```

- streak = 0 → 0.0
- streak = -1 → 0.2
- streak = -3 → 0.6
- streak = -5 и ниже → 1.0
- streak > 0 → 0.0 (положительный streak не даёт бонуса)

Делитель 5 — нормализация; streak -5 считается "максимально плохим".

#### decayFactor (вес 0.30)

Приоритет давно не показанных вопросов (кривая забывания).

```dart
decayFactor = lastShownAt != null
    ? clamp(daysSinceLastShown / 30, 0, 1)
    : 1.0;
```

- Показан сегодня → ~0.0
- Показан 15 дней назад → 0.5
- Показан 30+ дней назад → 1.0
- **Никогда не показан (null)** → 1.0 (максимальный приоритет)

Порог 30 дней — вопрос, не показанный месяц, считается полностью "забытым".

#### newFactor (вес 0.20)

Приоритет малоизученных вопросов (мало показов).

```dart
newFactor = 1 - clamp(totalShown / 5, 0, 1);
```

- 0 показов → 1.0
- 2 показа → 0.6
- 5+ показов → 0.0

Порог 5 — вопрос, показанный менее 5 раз, считается "новым".

#### frequencyFactor (вес 0.15)

Приоритет вопросов с высоким процентом ошибок.

```dart
frequencyFactor = totalShown > 0
    ? totalIncorrect / totalShown
    : 0.5;
```

- Все ответы правильные → 0.0
- 50% ошибок → 0.5
- Все ответы неправильные → 1.0
- **Нет статистики** → 0.5 (нейтральное значение — не штрафуем и не бонусим)

#### noise

Лёгкая рандомизация для разнообразия при близких score.

```dart
noise = random.nextDouble() * 0.05; // 0–5%
```

### Веса

```dart
static const _wStreak = 0.35;
static const _wDecay  = 0.30;
static const _wNew    = 0.20;
static const _wFreq   = 0.15;
```

Максимальный суммарный score без шума = 1.0. С шумом — до 1.05.

### Примеры расчёта

| Сценарий | streak | decay | new | freq | score |
|---|---|---|---|---|---|
| Новый вопрос, никогда не показан | 0.0 | 1.0 | 1.0 | 0.5 | 0.0 + 0.30 + 0.20 + 0.075 = **0.575** |
| streak=-3, показан 20 дней назад, 10 показов, 70% ошибок | 0.6 | 0.67 | 0.0 | 0.7 | 0.21 + 0.20 + 0.0 + 0.105 = **0.515** |
| streak=-5, показан вчера, 3 показа, 100% ошибок | 1.0 | 0.03 | 0.4 | 1.0 | 0.35 + 0.01 + 0.08 + 0.15 = **0.59** |
| streak=+5, показан 2 дня назад, 20 показов, 10% ошибок | 0.0 | 0.07 | 0.0 | 0.1 | 0.0 + 0.02 + 0.0 + 0.015 = **0.035** |
| Без статистики вообще | 0.0 | 1.0 | 1.0 | 0.5 | **0.575** |

### Ключевые свойства модели

- **Нет бинарного переключения** — все факторы работают одновременно
- **Новые вопросы (без статистики)** получают высокий score (~0.575) за счёт decay + new + freq
- **Давно забытые вопросы** приоритетнее недавно отвеченных даже при отрицательном streak
- **Хорошо выученные вопросы** (streak > 0, недавно показаны) получают минимальный score

## Выбор количества

Идентично классическому алгоритму:
```dart
count = random.nextInt(mixupMax - mixupMin + 1) + mixupMin
```

Берутся первые `count` кандидатов по убыванию score, помечаются `isMixedIn: true`.

## Зависимости

- `MixupCandidateLoader` — общая логика загрузки кандидатов и статистики
- `QuestionStatsEntity` — модель статистики (поля: streak, totalShown, totalCorrect, totalIncorrect, lastShownAt)

## Переключение алгоритмов

Выбор между `classic` и `scoring` хранится в `SettingsDto.mixupAlgorithm` (enum `MixupAlgorithm`), управляется через `MixupBloc`, фабрика в `TinderTestFlow` создаёт нужную реализацию `IQuestionMixupService`.
