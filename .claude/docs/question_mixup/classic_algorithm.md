# Классический алгоритм подмешивания (QuestionMixupService)

## Расположение

`lib/feature/tinder_test/domain/mixup/question_mixup_service.dart`

## Назначение

Подмешивает вопросы из других тестов той же группы в текущий тест. Цель — повторение ранее изученного материала в контексте текущего теста.

## Получение кандидатов

1. По `testId` находим все группы, в которые входит тест (`GroupsDatabase.getGroupIdsByTestId`)
2. Для каждой группы получаем все тесты (`GroupsDatabase.getTestIdsByGroupId`)
3. Исключаем текущий тест
4. Загружаем карточки из оставшихся тестов (`ICardRepository.getCardsByTestId`)
5. Фильтруем дубликаты — карточки, чей нормализованный ключ (`front + back`) совпадает с карточками основного теста
6. Загружаем статистику для кандидатов (`IQuestionStatsRepository.getStatsByKeys`)

## Стратегия сортировки

Алгоритм выбирает одну из двух веток сортировки на основе **глобального** признака:

```
hasNegativeStreak = statsMap.values.any((s) => s.streak < 0)
```

### Ветка A: есть хотя бы один вопрос с отрицательным streak

Приоритет отдаётся "сложным" вопросам:

| Приоритет | Поле | Направление | Смысл |
|---|---|---|---|
| 1 | `streak` | по возрастанию | Самые отрицательные streak первые |
| 2 | `totalIncorrect` | по убыванию | При равном streak — больше ошибок выше |

Вопросы **без статистики** уходят в конец списка.

### Ветка B: все streak >= 0 (или статистики нет)

Приоритет отдаётся давно не показанным и новым вопросам:

| Приоритет | Поле | Направление | Смысл |
|---|---|---|---|
| 1 | наличие статистики | без статистики первые | Новые вопросы приоритетнее |
| 2 | `lastShownAt` | по возрастанию (null первые) | Давно показанные выше |
| 3 | `totalShown` | по возрастанию | Реже показанные выше |

## Выбор количества

```dart
count = random.nextInt(mixupMax - mixupMin + 1) + mixupMin
```

Берутся первые `count` карточек из отсортированного списка, каждая помечается `isMixedIn: true`.

## Известные ограничения

1. **Бинарность** — один вопрос с отрицательным streak переключает стратегию для ВСЕХ кандидатов
2. **Забывание не учитывается в ветке A** — `lastShownAt` игнорируется при наличии отрицательных streak'ов
3. **Нулевая статистика в ветке A** — вопросы без статистики (самые неизученные) уходят в конец, хотя они наименее знакомы пользователю

## Зависимости

- `ICardRepository` — загрузка карточек по testId
- `GroupsDatabase` — связь тестов и групп
- `IQuestionStatsRepository` — статистика ответов
- `QuestionKeyNormalizer` — нормализация ключа вопроса (`front + back`, lowercase, trim, collapse spaces)

## Модель статистики (QuestionStatsEntity)

```
id, questionKey, frontText, backText,
streak          — серия: +1 за правильный, -1 за неправильный
totalCorrect    — всего правильных
totalIncorrect  — всего неправильных
totalShown      — всего показов
lastAnsweredAt  — дата последнего ответа (nullable)
lastShownAt     — дата последнего показа (nullable)
createdAt, updatedAt
```
