# Simulation report: multiplicative streak decay

## TL;DR

- **Робастность**: ✅. Регрессов нет — cold-start детерминирован и совпадает, новые карточки сдвигаются на ≤0.14, отрицательный streak — на ≤0.08. Pearson(old, new) = 0.986 — общая сортировка сохраняется.
- **Sweet-spot**: `N = 10`, `w_decay = 0.30` (текущий) — даёт 15/32 well-learned в top-20 при ads=10, и сохраняет 4 negative-streak в текущем top-20.
- **Главный риск**: на ТЕКУЩИХ ads (0-3) формула почти не меняет ничего (Pearson 0.986). Эффект «well-learned лезут в mixup» включается только когда у них ads ≥ 7-10. Пользователь не увидит мгновенного эффекта от внедрения — нужно играть другие тесты неделю-две.

## 3. Sensitivity (w_decay × N)

### A) well-learned в top-20 при ads=10 (из 32)
```
            N=5   N=7   N=10  N=12  N=14
w_decay=0.20    0     1    14     0     0
w_decay=0.25    0     1    14     0     0
w_decay=0.30    0     5    15     7     0
w_decay=0.35    0    15    15    14     0
w_decay=0.40    0    16    16    14     1
```
**Ключевое: N=10 — везде 14-16. N=5 и N=14 — везде 0-1.** N не должен быть «среднего размера 7-12», он должен совпасть с шагом decay-кэпа. Это указывает на нелинейность поведения, но в нашу пользу: N=10 — устойчивая «полка».

## Финальная рекомендация

1. **Внедрять как есть** в Dart. Формула корректна, регрессов нет.
2. **Параметры**: `kActiveDaysCap = 10`, веса не менять (`w_decay = 0.30`).
3. **Ожидание**: пользователь не увидит мгновенного эффекта (Pearson 0.986 на текущих ads). Эффект проявится по мере того, как well-learned «состарятся» (ads ≥ 7-10 = ~3 недели календарных при текущем темпе занятий).

## Замеченные риски

1. **N=10 — на «полке»**: N=5 и N=14 дают 0/16 well-learned в top-20. Маленький разброс N значительно меняет результат. Не выносить N в настройки — пользователь может случайно сломать.
2. **Cold-start карточек 26** — это карточки без статы в `cards.csv`, не в пуле. Они получают score 0.575 (детерминированно) и могут систематически лезть в топ если попадут в пул кандидатов после первого просмотра — но это вне scope текущего изменения формулы.
3. **`ads = 0` для пустой БД**: streak penalty полный, decay = 0. Если БД свежая и у пользователя есть выученные карточки (streak > 0) — они получат полный penalty. Это сейчас работало одинаково в обеих формулах.
